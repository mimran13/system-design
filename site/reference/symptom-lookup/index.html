    Symptom → Concept Lookup¶

    Spot a problem at work, find the right concept fast. This page is diagnostic-first: each row is a real-world symptom you'd actually say out loud, mapped to the page that explains the underlying concept and what to do about it.

    Use Cmd+F / Ctrl+F to search by symptom keyword.

    Reliability and outages¶

    If you see this... Likely concept What to read
    One slow downstream service is taking down our whole stack Cascade failure → Circuit Breaker Circuit Breaker
    200 threads stuck waiting on a 30-second timeout Missing timeouts / no bulkhead Retry & Timeout, Bulkhead
    Service A goes down → Service B goes down → Service C goes down Cascading failure across services Circuit Breaker, Bulkhead
    Service is "healthy" per monitoring but users see errors Gray failure Failure Modes
    Brief outage → 1M clients all retry simultaneously → outage stays Thundering herd Backoff Strategies
    Retries are making things worse, not better Need exponential backoff with jitter Backoff Strategies, Retry & Timeout
    One service's failure cascades to many No bulkhead isolation Bulkhead
    "We had an outage and only one team could deploy" Distributed monolith Anti-Patterns
    Outage when one shared component fails (auth, DNS) Single point of failure Availability & Reliability, Failure Modes
    Pages stop responding at exactly 3 AM nightly Likely cron job spike or DB maintenance window Failure Modes, Hot Partitions
    Outage post-mortem says "load balancer health check passing but pods broken" Health check too shallow Failure Modes

    Performance¶

    If you see this... Likely concept What to read
    Reads are fast but writes lag behind LSM trade-offs, or write-heavy workload on B-tree DB Storage Engine Internals
    Queries that touched 100 rows now touch 10M Missing index, full table scan Database Indexes
    Adding more servers doesn't add proportional throughput Hitting Amdahl's / USL ceiling Throughput Limits
    One shard at 90% CPU, others at 5% Hot partition / hot key Hot Partitions & Hotspots
    One key in Redis serves 50% of traffic Hot key Hot Partitions, Caching Patterns
    Latency P99 is 10× P50 Tail latency, fan-out amplification Latency vs Throughput, Throughput Limits
    Database CPU pegged but queries look simple N+1, missing index, or vacuum needed Database Indexes
    Cold start of a new instance is unbearably slow Missing warm-up, JIT, or page cache OS Concepts, Memory Hierarchy
    Throughput plateaus far below network bandwidth TCP window / BDP problem TCP/UDP Deep Dive
    First HTTP request slow, rest fast TCP handshake + TLS not pooled TCP Deep Dive, TLS
    Request fans out to 50 services; P99 is awful Fan-out tail latency Latency vs Throughput
    Reading code is fine but writing it scrolls forever in editor Cache lines / file size — different problem

    Data correctness and concurrency¶

    If you see this... Likely concept What to read
    Two operations both ran; we expected only one Need idempotency, dedup keys Idempotency
    Webhooks delivered twice; processing applied twice Idempotency keys missing Idempotency
    Counter race condition: two increments → only one stuck Lost update Database Transactions & Isolation, Concurrency
    Negative balances appearing under load Read-then-write race Isolation Levels
    Constraint violated (e.g., double-booked) under load Write skew at Snapshot Isolation Isolation Levels
    Different users see different values for "the same" data Eventual consistency in action Consistency Models, CAP Theorem
    User updates profile, refreshes page, old value back Read-after-write inconsistency Consistency Models
    Database replication lag = 10s during traffic spike Read replica lag Replication, Read Replicas
    "We need exactly-once message delivery" Spoiler: usually you need idempotency, not actual exactly-once Idempotency, Exactly-Once Semantics
    Distributed transaction across services is painful Saga pattern is usually the answer Saga Pattern
    Order placed but payment never charged (or vice versa) Multi-step transaction without saga Saga Pattern, Outbox Pattern
    Database write succeeded but message wasn't published Dual-write problem Outbox Pattern
    Different replicas of same record disagree Eventual consistency, conflict resolution CRDTs, Quorum
    "Schema change broke 5 services" Service coupling at the data layer Coupling & Cohesion at Service Boundaries

    Scale and capacity¶

    If you see this... Likely concept What to read
    Adding more web servers doesn't help; database is the bottleneck Read scaling needed Read Replicas, Caching Strategies
    Single Postgres at 80% capacity, growing Need to shard, replicate, or both Sharding, Read Replicas
    One table is 90% of database size Should be sharded or moved to specialised store Sharding, Time-Series DBs
    Connection limit reached on database under load Connection pool exhaustion Connection Pooling
    Adding a new node to a cluster moves nearly all keys Naive hash mod N partitioning Consistent Hashing
    Need to add capacity without downtime Consistent hashing + virtual nodes Consistent Hashing, Sharding
    Multi-tenant SaaS, one giant tenant slows everyone Noisy neighbour Multi-Tenancy, Hot Partitions
    Black Friday traffic 10× normal for 4 hours Auto-scaling won't be fast enough; pre-warm Capacity Planning
    Service is fast under load test but breaks in production Real traffic patterns differ; queuing theory Queuing Theory
    90% CPU feels like the system is broken Queuing theory: above 70% utilisation, latency spikes Queuing Theory

    Caching¶

    If you see this... Likely concept What to read
    Cache miss storm after deploy / restart Cold cache, warm-up needed Caching Strategies, Cache Patterns
    Cache stampede: 1000 requests rebuild same cache entry Lock or single-flight needed Cache Patterns & Pitfalls
    Stale data shown after writes Cache invalidation strategy Cache Invalidation
    Cache hit rate is 30%, was 90% last week Working set grew or eviction policy wrong Eviction Policies
    Need cache to be consistent across N app servers Distributed cache Distributed Caching
    1M servers all read same Redis key constantly Hot key in cache Hot Partitions, Cache Patterns

    Architecture and team¶

    If you see this... Likely concept What to read
    5-person team running 30 microservices, drowning in ops Premature microservices Anti-Patterns, Modular Monolith
    Every PR touches multiple services Distributed monolith Anti-Patterns, Coupling at Service Boundaries
    Services share a database / read each other's tables Service coupling at data layer Coupling at Service Boundaries
    Onboarding a new engineer takes 6 months Big ball of mud / no boundaries Anti-Patterns
    Hard to know what changes when "core service" is touched God service Anti-Patterns
    50 engineers, one codebase, deploys are scary Need modular boundaries or split Modular Monolith, Microservices Patterns
    Migrating from monolith to microservices, frozen for months Big-bang rewrite (don't) Strangler Fig
    Domain logic mixed with database / framework code Tight coupling, no hexagonal Hexagonal Architecture
    "It works on my machine" — env config drift Twelve-factor violation Twelve-Factor App
    Mobile app and web app need different API shapes Backend for Frontend BFF
    Need audit trail for every state change Event sourcing fits Event Sourcing, CQRS+ES
    Read load 100× write load, single model serving both CQRS opportunity CQRS
    Need to rebuild a "view" of data with new fields Event sourcing replay Event Sourcing
    12 services in a workflow, all subscribing to events, lost track of flow Need orchestration, not choreography Choreography vs Orchestration
    Long-running workflow (waits, human approval, retries across days) Workflow engine fits (Temporal, Step Functions) Choreography vs Orchestration
    Users in EU complain of slow load times Multi-region or edge Multi-Region, Edge Architecture
    Need GDPR data residency Per-tenant region Multi-Region, Multi-Tenancy

    Database choice¶

    If you see this... Likely concept What to read
    Querying with complex joins across many tables Relational database Relational Databases
    Simple lookups by ID, ultra-low latency Key-value store Key-Value Stores, SQL vs NoSQL
    Document with nested arbitrary fields, no fixed schema Document store Document Stores
    Append-only events with timestamp queries Time-series database Time-Series Databases
    Full-text search, ranking, fuzzy matching Search engine Search Engines
    Wide rows, billions of records, write-heavy Wide-column store (Cassandra, ScyllaDB) Wide-Column Stores
    Vector similarity search for AI / embeddings Vector database Vector Databases
    Highly connected data (social graph, recommendations) Graph database Graph Databases
    Need analytics on petabytes Data warehouse, columnar Data Warehousing
    "We picked NoSQL because it's faster" — and it isn't Wrong tool for the job SQL vs NoSQL
    Need ACID at scale, multi-region NewSQL (Spanner, CockroachDB) NewSQL

    Networking and APIs¶

    If you see this... Likely concept What to read
    Need streaming bidirectional connection (chat, live updates) WebSockets or SSE WebSockets & SSE
    Need very low latency RPC between services gRPC gRPC
    Frontend wants to specify exactly which fields to fetch GraphQL GraphQL
    External integration that pushes events to us Webhooks Webhooks
    API change broke 50 customers Breaking change without versioning API Versioning, API Versioning at Architecture Level
    Need to abuse-proof an endpoint Rate limiting Rate Limiting
    User makes 1000 requests/sec Token bucket / leaky bucket Rate Limiting
    TLS handshake taking 200+ms TLS 1.2 → upgrade to 1.3 or use 0-RTT resumption TLS and Certificates
    Mobile users on flaky networks see lots of failures TCP head-of-line blocking; consider QUIC/HTTP/3 TCP/UDP Deep Dive, HTTP Versions

    Messaging¶

    If you see this... Likely concept What to read
    Producer outpacing consumer; queue depth growing Backpressure or scale consumer Backpressure
    Want fan-out to many subscribers Pub/sub Pub/Sub
    Want replay-able event log Kafka Event Streaming, Kafka
    Need exactly-once message processing Probably need idempotent consumers Idempotency, Exactly-Once
    Order matters in messages Single partition / consumer Event Streaming, Kafka
    Lost messages when broker restarts Need acknowledgement / persistence config Message Queues, Kafka

    Security¶

    If you see this... Likely concept What to read
    Need user login OAuth 2.0 / OIDC OAuth 2.0 & JWT
    API needs auth between services mTLS or signed JWTs TLS, Zero Trust
    Secrets in environment variables / config files Secrets manager needed Secrets Management
    Public S3 bucket / open security group found in audit IaC scanning needed Secrets in IaC, Testing IaC
    Network breach in one service spread laterally No zero trust / no segmentation Zero Trust
    API hammered with bot requests Rate limiting + bot detection at edge Rate Limiting, Edge Architecture
    Compliance audit: "show every action by every user" Event sourcing or audit log Event Sourcing

    Production looks different from staging Twelve-factor / config drift Twelve-Factor App, IaC
    Manual changes in cloud console; nobody knows what's deployed Need IaC IaC Fundamentals
    "Cluster state doesn't match Git" Drift Drift Detection
    Deploys are slow / scary No automation, big-bang releases CI/CD Fundamentals, Deployment Strategies
    Bug in production, can't roll back without deploy Need feature flags Branching Strategies, Release Management
    1% of users hit a new bug right after deploy Canary deploy + automated rollback Deployment Strategies, Progressive Delivery

    Outage took 2 hours to diagnose Observability gap Observability, Distributed Tracing
    Pages firing all night, can't tell what's actually broken Alerting too noisy / too coarse Alerting, SLO
    "We met 99.9% uptime" — but customers complain SLO doesn't match user experience SLI, SLO & SLA

    How to use this page¶

    1. You hit a problem at work. You don't know what it's called.
    2. Cmd+F the symptom. Search by the words you'd actually use ("slow", "stuck", "race", "lag", "sharding", "double-charged").
    3. Click through to the concept page. Read the "What it is" + "When to use it" sections — that's enough to act on.
    4. Implementation later. The concept page has code, config examples, library links if you decide to apply.

    This page is intentionally incomplete by design. It's the symptoms most engineers run into; not every page is here. If you can't find your symptom, browse by topic in the left sidebar.
